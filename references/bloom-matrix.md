## 一、布鲁姆6×4矩阵（安德森修订版）

### 认知过程维度（6层）+ 知识维度（4类）交叉

```
               事实性知识  概念性知识  程序性知识  元认知知识
记忆 (Remember)    ✅         ✅         ✅         ✅
理解 (Understand)  ✅         ✅         ✅         ✅
应用 (Apply)       ✅         ✅         ✅         ✅
分析 (Analyze)     ✅         ✅         ✅         ✅
评价 (Evaluate)    ✅         ✅         ✅         ✅
创造 (Create)      ✅         ✅         ✅         ✅
```

### 各知识类型定义

| 知识类型 | 定义 | 教学示例 |
|---------|------|---------|
| **事实性知识** | 孤立的信息要素，术语/事实/细节 | 年份、人名、公式、定义 |
| **概念性知识** | 分类与类别、原理与通则、理论与结构 | 大概念、因果关系、系统模型 |
| **程序性知识** | 怎么做、方法、技能、技术 | 解题步骤、实验流程、写作方法 |
| **元认知知识** | 策略性知识、认知任务知识、自我知识 | 学习策略、自我监控、反思 |

## 二、行为动词表（按层级）

### 记忆层
```
列出 / 背诵 / 复述 / 识别 / 回忆 / 定义 / 命名
```
示例："学生能**列出**光合作用的三个必要条件。"

### 理解层
```
解释 / 总结 / 举例 / 分类 / 推断 / 比较 / 说明
```
示例："学生能用自己的话**解释**细胞分裂的过程。"

### 应用层
```
执行 / 实施 / 使用 / 演示 / 操作 / 求解
```
示例："学生能**运用**勾股定理计算实际距离。"

### 分析层
```
区分 / 组织 / 归因 / 拆解 / 比较 / 鉴别
```
示例："学生能**区分**事实性描述和作者观点。"

### 评价层
```
检查 / 评判 / 论证 / 辩护 / 批评 / 评估
```
示例："学生能**评价**两个实验方案各自的优劣。"

### 创造层
```
生成 / 计划 / 发明 / 设计 / 建构 / 创作
```
示例："学生能**设计**一个实验验证自己的假设。"

## 三、ABCD目标写法

```
A (Audience) 受众 → X年级学生
B (Behavior) 行为 → 通过可观察的动词描述
C (Condition) 条件 → 在什么情境下/用什么工具
D (Degree) 程度 → 达到什么标准
```

**示例**：

```
[小学五年级学生]（A）
能在不借助计算器的情况下（C）
计算两位小数的加减法（B）
正确率达到90%以上（D）
```

## 四、AI时代目标重置指南

### 核心原则（来自AIED知识库）

> "课程的核心应当是行动与表现，而非单纯的知识积累。"
> —— Grant Wiggins, UbD创始人

> "当机器可以代替我们'创造'时，学习不再是通往金字塔顶端的苦行，
> 而是在大脑与外脑、低阶与高阶之间，那场永无止境、螺旋上升的对话。"
> —— 反向布鲁姆框架分析

### 每个目标的自检清单

设计每个教学目标时，务必自问：

```
□ 这个目标AI能做到吗？
  → 如果AI完全能做到 → 目标需要升级到"人类判断+应用AI结果"
  → 如果AI部分能做到 → 明确"AI可以辅助什么，人类必须做什么"

□ 这个目标指向"知识记忆"还是"可迁移的理解"？
  → 知识记忆 → 考虑是否有必要提升到概念理解或应用层面

□ 学生完成这个目标后，能做什么在AI时代有价值的事？
  → 批判性思维、系统思维、创造、伦理决策 → 优先

□ 这个目标是否能引发"认知摩擦"？
  → AI轻松解决的问题不能作为高阶目标
  → 学生必须亲自经历的认知挣扎才是核心
```

### AI时代目标升级示例

| 传统目标 | AI时代升级版 |
|---------|-------------|
| "复述《岳阳楼记》的主要内容" | "比较AI生成的《岳阳楼记》解读和你的理解，指出差异并论证你的观点" |
| "计算圆的面积" | "设计一个花坛方案（给定预算和场地约束），用AI辅助计算成本，评估不同方案的可行性" |
| "记住光合作用的公式" | "设计一个实验证明光照强度对光合作用的影响，用AI模拟不同条件的结果，分析模拟与实际可能的差异" |

### 低阶认知 vs 高阶认知的AI分工

```
低阶认知（记忆、理解、简单应用）
  → 逐渐由外脑（AI）承担"结构化表征"任务
  → 教师设计时考虑：需要学生记住什么？可以借助AI什么？

高阶认知（分析、评价、复杂创造）
  → 成为内脑（人类）的"策展人"阵地
  → 教师设计时考虑：学生如何驾驭AI输出的知识流？
  → 评价重心从"你拥有多少知识"转向"你如何驾驭知识流"
```

### 反向布鲁姆给目标设计的启示

> 赵晓伟博士的"逆向重构课堂"（来源：AIED知识库）：

```
起点：不是教师预设的知识点讲授
而是：直接抛出"综合"或"评价"层级的高挑战性任务
      → 人为制造认知失调
      → 学生因基础认知结构缺失陷入困境
      → 带着明确的认知需求，主动回溯到低阶层级
      → "按需下潜"补缺
      → 重新浮升至高层级完成跃迁
```

适用场景：探究课/项目式学习/单元整合课

## 五、核心问题设计

### 好问题的标准

```
- 开放而非封闭（不是"是/否"可答）
- 指向大概念而非孤立事实
- 引发持续思考而非一次性回答
- 与学生经验有连接点
```

### 示例

| 学科 | 弱问题 | 强核心问题 |
|-----|-------|-----------|
| 语文 | "作者用了什么修辞手法？" | "语言如何塑造我们的身份认同？" |
| 数学 | "勾股定理的公式是什么？" | "我们如何确定一个位置是唯一的？" |
| 科学 | "光合作用需要什么条件？" | "生命如何把阳光变成自己的能量？" |
| 历史 | "辛亥革命发生在哪一年？" | "社会变革如何发生——流血还是改良？" |

## 六、差异化目标设计模板

```
基础目标（全体达标）：
  布鲁姆层级：[记忆/理解]
  行为描述：[最低要求]

标准目标（大部分学生）：
  布鲁姆层级：[理解/应用]
  行为描述：[正常要求]

拓展目标（学有余力）：
  布鲁姆层级：[分析/评价/创造]
  行为描述：[更高要求]
```

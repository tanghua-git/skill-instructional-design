## 一、CREATE 六维备课自检框架

### 六维定义

| 字母 | 维度 | 核心问题 | 检查要点 |
|------|------|---------|---------|
| **C** | 用户洞察 | 我了解学生的起点和卡点吗？ | 学情分析是否具体？是否有数据支撑？ |
| **R** | 真实任务 | 学生知道为什么学这个吗？ | 是否有真实任务/情境驱动？ |
| **E** | 学习旅程 | 学习路径是否清晰、有逻辑？ | 环节之间是否有因果联系？ |
| **A** | 锚定目标 | 每个环节都指向目标吗？ | 活动↔目标是否一一对应？ |
| **T** | 评价工具 | 我怎么知道学生学会了？ | 是否有可操作的评价方式？ |
| **E** | 教学包 | 其他老师拿到我的设计能用吗？ | 是否完整？是否有操作细节？ |

### 自检问题清单

```
C-用户洞察：
  □ 我是否预判了学生可能的迷思概念？
  □ 我是否考虑了我用什么具体方法来验证这些预判？
  □ 我的设计是否从学生的角度出发，而不是从知识点的角度？

R-真实任务：
  □ 学生学了这节课后能解释或解决什么真实问题？
  □ 这个任务是真的有意义，还是"看起来真实"？
  □ 学生回答"我为什么要学这个"时，我给出的理由是什么？

E-学习旅程：
  □ 从"不回到"到"会"，学生经历了几个环节？
  □ 这些环节的顺序是否有逻辑依据？（认知负荷/图式理论）
  □ 是否有明确的从浅到深的递进？

A-锚定目标：
  □ 每个活动能否回答"这个活动让学生学到什么"？
  □ 答不出的话，这个活动有必要存在吗？

T-评价工具：
  □ 怎么知道学生是否达成了每个目标？
  □ 评价方式是否与目标的层级匹配？（记忆→问答，应用→任务）
  □ 是否有前测和后测的对比？

E-教学包：
  □ 我的设计对另一位老师来说是否可重复？
  □ 是否有明确的板书设计？
  □ 是否有教学反思的预留空间？
```

## 二、五维一致性审查矩阵

### 审查范围

```
① 目标 ↔ 活动：每个活动是否指向至少一个目标？
② 目标 ↔ 评估：每个目标是否有对应的评估方式？
③ 活动 ↔ 评估：评估是否在活动中自然发生？
④ 动机 ↔ 活动：活动是否有自主/胜任/归属设计？
⑤ 元认知 ↔ 活动：关键节点是否有元认知提示？
```

### 审查表模板

```
一致性审查矩阵

目标1 (记忆·事实) | 活动1 [✅] | 活动2 [✅] | 活动3 [⚠️无指向]
目标2 (理解·概念) | 活动1 [⚠️弱关联] | 活动2 [✅] | 活动3 [✅]
目标3 (应用·程序) | 活动1 [❌] | 活动2 [✅] | 活动3 [✅]

评估检查：
  目标1 → 前测 [✅] / 后测 [✅]
  目标2 → 前测 [✅] / 后测 [✅]
  目标3 → 前测 [⚠️缺] / 后测 [✅]

动机检查：
  活动1: 自主[✅] / 胜任[✅] / 归属[⚠️缺]
  活动2: 自主[⚠️缺] / 胜任[✅] / 归属[✅]
  活动3: 自主[✅] / 胜任[✅] / 归属[✅]

元认知检查：
  活动前 [✅] / 活动中 [⚠️缺] / 活动后 [✅]
```

## 三、ICAP档位合理性检查

### 判断规则

```
新授课：
  - 开始阶段P/A档是合理的（学生需要先接收信息）
  - 后期应逐步升至C/I档
  - × 全程P档（被动听讲无产出）
  - × 一上来就I档（学生还不会就开始互动）

复习课：
  - C档为主（建构概念图/结构化知识）
  - × 全程A档（只做重复练习无建构）

练习课：
  - A→C档为主（练习+解释为什么）
  - × 全程P档（只看示范不做）

探究课：
  - C→I档为主（发现/建构/互动）
  - × 表面C档实则A档（分组但各说各话无共建）

讲评课：
  - C档为主（分析错因/自我反思）
  - × 教师从头讲到尾无学生参与
```

## 四、差异化覆盖检查

```
检查清单：
  □ 是否有基础路径？— 最低要求/显性支持
  □ 是否有标准路径？— 正常设计
  □ 是否有拓展路径？— 更高挑战
  
  如果某类学生完全无设计 → 标记为❌
```

## 五、AI教案五点评估

> 来源：《教师AI备课正确打开方式》（kb1）

```
1. 是否包含学生困难分析？（不是只有教学流程）
2. 是否有贯穿全课的核心问题？
3. 是否有层次分明的问题链？
4. 每个活动是否回扣目标？（不是为活动而活动）
5. 是否有检测理解的工具？
```

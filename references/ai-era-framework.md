## 一、AI评估三分类细则

### 三类评估的课程安排建议

```
课程层面：三类评估融合，学期内有序排序

学期初 → 无AI评估（摸底独立能力）
学期中 → AI辅助评估（过渡期）
学期末 → AI整合评估（综合能力）
最终 → 三者混合（平衡独立与协作能力）

⚠️ 不要全无AI（脱离现实）也不要全AI辅助（能力退化）
```

### 14种AI整合评估方法（精选7种）

```
1. 过程记录法
   学生记录每一步操作（含AI使用），展示思考过程而非最终结果

2. 作品+答辩法
   提交作品后口头答辩，考察真实理解而非AI产出

3. 情境化任务（GRASPS增强版）
   真实情境+具体角色+可验证的成果标准

4. 多模态证据
   文字+图表+口头+实操，单一证据不足以证明理解

5. 迭代追踪
   比较初稿/终稿/修改记录，可见学生的思维演进

6. 反思声明
   要求学生反思"AI帮了我什么""我还需要学什么"

7. 协作评估
   团队完成+个人贡献自评+同伴互评
```

## 二、AI时代目标重置细则

### 七问自检

设计每个目标时，回答以下七个问题：

```
□ 这个目标AI能做到吗？
  如果AI完全能做到 → 目标需要升级
  如果AI部分能做到 → 明确人机分工

□ 这个目标指向"记忆"还是"理解"？
  记忆目标是否必要？能否用其他方式替代？

□ 学生完成这个目标后，能做什么在AI时代有价值的事？
  批判性思维？系统思维？创造？伦理决策？

□ 这个目标是否能引发"认知摩擦"？
  AI能轻松解决的问题不能作为高阶目标
  学生"挣扎"的过程才是核心学习

□ 这个目标的成果是否可迁移？
  离开这个班级/这学期/这个学校还能用吗？

□ 这个目标的评价方式能否抵抗AI的直接回答？
  如果是搜索即得答案 → 需要改形式

□ 学生是否知道这个目标为什么值得学？
  不是"考试要考"而是"因为你需要这个能力"
```

### 目标类型转换表

| 传统目标 | 问题 | AI时代目标 |
|---------|------|-----------|
| "记住××公式" | AI能直接给出 | "在什么情境下选择这个公式？为什么不是另一个？" |
| "列举××特征" | AI能快速生成 | "比较AI生成的特征和你的观察，谁的更准确？"（评价） |
| "背诵××段落" | AI能即时提供 | "用自己的话重构核心意思，并和AI版本对比分析差异"（分析+创造） |
| "按步骤操作" | AI能模拟 | "设计一个更优的操作流程，并论证为什么更好"（创造+评价） |

## 三、教育学人AI能力谱系L0-L5

### 定义

| 级别 | 名称 | 教师投入 | AI能力 |
|------|------|---------|--------|
| L0 | 辅助生成 | 逐字校对纠错 | 生成草稿 |
| L1 | 逻辑推理 | 审核逻辑与对齐 | 给结构性方案 |
| L2 | 工具联动 | 调度与门禁 | 查/算/整/导出 |
| L3 | 智能代理 | 设定目标+验收 | 闭环交付 |
| L4 | 场景专家 | 设计系统 | 校本生产线 |
| L5 | 智慧共生 | 关注发展质量 | 数字孪生伙伴 |

### 对本Skill的启示

```
Skill设计的目标：帮助教师从L0/L1提升到L2/L3
  即：不是让AI逐字生成教案（L0），
      而是让AI辅助教师思考设计逻辑（L1），
      最终教师能设定目标→调度AI→验收输出（L3）
```

## 四、 五层认知框架（课程重构中心报告）

> 来源：美国课程重构中心2026报告（AIED知识库）

### 五层

```
L5 显式教授认知元能力 → 学生知道如何学习
L4 跨域综合意义建构 → 学生能整合不同知识
L3 深化概念理解 → 学生能解释"为什么"
L2 策略性自动化程序技能 → 该自动化的交给自动化
L1 保护基础人类能力 → 哪些能力不能外包
```

### 教学设计应用

```
在目标设计中自问：
  - 这个目标在L1-L5的哪个层级？
  - 是否过多集中在L1/L2？（基础记忆和技能）
  - 是否需要向L3/L4/L5提升？
```

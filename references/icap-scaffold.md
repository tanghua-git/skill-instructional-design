## ⚠️ 关键警告：教法学法 ≠ 教学策略

> 大多数老师的教案，教法学法都写错了。
> 他们写的是分工，不是工具；是阵型，不是战术。
> 
> — 哈士奇聊教学设计，第179篇

### 核心区分

```
教法学法：
  → 解决"谁来干"（课堂分工）
  → 例：讲授法、小组合作、探究式学习、任务驱动
  → 问题：只搭了舞台，没写剧本

教学策略：
  → 解决"怎么干"（具体工具）
  → 例：发言筹码、拼图阅读、故事山、问题矩阵
  → 价值：学生拿到就会用的具体工具
```

### 教师最常见的错误

```
❌ "本节课采用小组合作与探究式教学" 
   → 这是套话，放哪节课都能用
   → 只写了姿态，没写动作

✅ "运用发言筹码机制确保全员参与，
   配合问题矩阵引导学生从低阶走向高阶提问"
   → 这才是真正能用的策略
```

### 三条获取策略的路径

```
路径1️⃣ 向上抽
  看到一个好活动（如"角色扮演模拟星体运行"）
  → 别只记教法名"情境模拟法"
  → 抽出底层策略："具身认知"
  → 教法跨不了学科，策略可以

路径2️⃣ 跨界偷
  最好的策略往往不在教育学课本里：
  "故事山" → 偷自好莱坞三幕剧
  "鱼骨图" → 偷自企业质量管理
  "六顶思考帽" → 偷自商业咨询
  "SWOT分析" → 偷自战略管理

路径3️⃣ 命名封装
  画不出来的策略就只是一团感觉
  → 给它起个名字
  → 画成图表/模板
  → 写清楚什么时候用它
  → 让它变成工具箱里随时能拿出来的东西
```

### 写教案的自检

```
在写下"合作探究"之前，先问自己：
  当学生面对这个难点的时候，
  我递给他的那把具体的工具，到底长什么样？

如果答不上来 → 你的教法学法栏写的是套话
```



## 一、ICAP认知投入框架

### 四档定义

| 档位 | 名称 | 学生外显行为 | 学习效果 | 升档策略 |
|------|------|-------------|---------|---------|
| **P** | 被动 | 接收信息，不做其他事 | 基准线 | → 加"用自己的话总结" |
| **A** | 主动 | 重复/复制/操练 | P的1.5倍 | → 加"解释为什么" |
| **C** | 建构 | 产生新内容（超出给出的） | P的2倍 | → 加"同伴互评" |
| **I** | 互动 | 对话+轮流建构 | P的3倍 | 最高档 |

### 常见活动升档对照表

| 活动 | P挡 | A挡 | C挡 | I挡 |
|------|-----|-----|-----|-----|
| **听讲** | 只听 | 边听边划重点 | 听完用自己的话写总结 | 两人互讲互评总结 |
| **阅读** | 通读 | 划线+标注 | 画概念图 | 讨论后合作画概念图 |
| **做题** | 看示范题 | 照步骤做一遍 | 做完加"解释我的思路" | 互批+互相讲解 |
| **讨论** | 听别人讨论 | 记录别人的观点 | 提出自己的观点+理由 | 回应+反驳+补充别人 |
| **笔记** | 无 | 逐字抄板书 | 用自己的话重组 | 和同桌交换笔记互相补充 |

## 二、扶放有度模型

### 四阶段

```
我做，你看                   你独立做
（高结构高支持）         （零结构零支持）
    │                         │
    ▼                         ▼
[示范] → [引导] → [协作] → [独立]
    ↑                        ↑
  我们一起做                你做，我看
（中结构中支持）         （低结构低支持）
```

### 每个阶段的教学行为

| 阶段 | 教师行为 | 学生行为 | 适用时机 |
|------|---------|---------|---------|
| Focus Lesson（我做你看） | 清晰讲授/示范/定义概念 | 看/听/记 | 新概念初建 |
| Guided Instruction（一起做） | 提问/提示/纠错 | 尝试/回答/练习 | 学生开始尝试 |
| Collaborative（你做我看） | 观察/必要时介入 | 小组合作/互相讲解 | 基本理解已建立 |
| Independent（你自己做） | 彻底撤离 | 独立完成/展示 | 掌握已确认 |

### 关键原则

```
- 不能跳过前两个阶段直接进入第四阶段（这不是独立，这是认知超载）
- 支架存在的唯一目的是最终被拆除
- 但"最终要拆除"不等于"一开始就不搭"
- 讲授不代表灌输，指导不代表控制
- "什么时候扶"和"什么时候放"需要刻意练习
```

## 三、明线与暗线设计

### 概念

```
明线 = 学生在课堂上能直接体验到的（活动/情境/任务）
暗线 = 教学目标/逻辑线（学生可能感受不到，但教师心里清楚）
```

### 设计要点

```
明线设计标准：
  - 吸引学生参与（"这个好玩/有意思/跟我有关"）
  - 与学生的生活经验有连接点
  - 但明线本身不是目的，它是手段

暗线设计标准：
  - 时刻自问：我做这一切的目的是什么？
  - 这个活动到底对应哪个教学目标？
  - 学生现在很嗨，那个目标达成了吗？
```

### 明暗线对照表示例

| 环节 | 明线（学生体验） | 暗线（目标指向） |
|------|---------------|---------------|
| 导入 | "有没有被冤枉过？来讲讲" | 引出"公正"概念，激发认知冲突 |
| 任务1 | 分组讨论一个法庭案例 | 理解程序公正 vs 结果公正 |
| 任务2 | 设计自己的"班级公约" | 应用公正原则到真实情境 |
| 总结 | "你觉得哪种公平更重要？" | 评价不同公正观的适用边界 |

## 五、提问技术

### 三层分类

| 问题类型 | 目的 | 适用环节 | 等待时间 |
|---------|------|---------|---------|
| **收敛性问题**（检测记忆） | "这是哪一年？""公式是什么？" | 回顾/检测 | 3秒 |
| **发散性问题**（促进建构） | "你还能想到什么？""如果...会怎样？" | 探究/应用 | 5秒+ |
| **元认知问题**（促进反思） | "你是怎么想到这个方法的？""卡在哪？" | 反思/总结 | 5秒 |

### 追问链

```
学生回答后 → "为什么？" → 学生进一步解释
            → "还有呢？" → 学生补充更多
            → "这和XX有什么关系？" → 建立联系
            → "你确定吗？如果反过来呢？" → 检验理解
```

## 六、差异化活动设计模板

所有活动提供三级路径：

```
基础路径：
  - 步骤更细（更多支架）
  - 提供示例/模板
  - 任务复杂度更低

标准路径：
  - 正常设计

拓展路径：
  - 减少支架/增加自主性
  - 要求迁移到新情境
  - 增加评价/创造层要求
```

## 一、什么是元认知

### 定义

> 元认知 = 思考自己的思考
> 知道自己知道什么、不知道什么、怎么知道的、怎么学会的

### 两大成分

```
元认知知识（知道什么策略 + 什么时候用 + 自己适合什么）：
  - 陈述性知识："我知道解释和总结是不同的策略"
  - 程序性知识："我知道怎么用概念图整理知识"
  - 条件性知识："我知道什么时候该用概念图而不是死记硬背"

元认知调节（计划→监控→评估→调整）：
  - 计划：学之前想"我打算用什么方法？"
  - 监控：学之中想"我现在理解了吗？卡在哪？"
  - 评估：学之后想"我学会了吗？什么方法最有效？"
  - 调整：根据评估决定"下次我该怎么做？"
```

## 二、元认知嵌入教学的三个关键策略

### 策略1：出声思考（Think Aloud）

```
教师示范：
  教师朗读时，说出发出"我在想什么"——
  "哦，这里作者说……但我有个疑问……"
  "我刚才没看懂这段，所以我倒回去重读了一遍……"
  "这个角色说的话，让我联想到……"

学生练习：
  两人一组，一人解题/阅读时说出思考过程
  另一人听，结束后总结"他用了什么策略？卡在哪？"
```

### 策略2：反思卡（嵌入每个练习节点）

```
练习前（30秒）：
  "我打算用什么方法？"
  "这道题和上次做过的有什么相似/不同？"
  "我需要特别注意什么？"

练习中（随时）：
  "我现在理解了吗？"
  "卡在哪一步？"
  "我还能用什么其他方法？"

练习后（2分钟）：
  "这次我用了什么方法？有效吗？"
  "卡在哪？下次怎么避免？"
  "和上一次比，我进步了什么？"
```

### 策略3：自我提问清单（学科通用版）

```
解题前：
  □ 我理解这个问题吗？（用自己的话复述一遍）
  □ 这和以前做过的题有什么相似之处？
  □ 我需要什么信息？还需要什么？

解题中：
  □ 我现在在用的方法对吗？
  □ 卡住了 → 我可以尝试什么不同方法？
  □ 我的答案是合理的吗？（快速估算验证）

解题后：
  □ 我的答案对吗？怎么验证？
  □ 有没有更好的方法？
  □ 这道题教了我什么？我下次遇到类似的会怎么做？
```

## 三、元认知嵌入的活动设计

### 嵌入位置（不额外加时间，而是替换）

```
❌ 错误做法：课末忽然说"现在花5分钟反思"（流于形式）
✅ 正确做法：替换现有的低认知活动

替换示例：
  原来的"抄板书"→ 替换为"用自己的话写总结+说明为什么这么理解"
  原来的"做10道题"→ 替换为"做8道题+写2道题的解题思路分析"
  原来的"记住公式"→ 替换为"解释这个公式和之前那个公式的关系"
```

### 学科嵌入示例

```
数学："做完这道题后，用一句话总结解题关键"
语文："读完这段后，标记出你不确定的3个理解点"
英语："写完之后，圈出你最有信心的3个单词和最没信心的3个"
科学："实验前写下你的预测，实验后对比差异"
历史："你认为这个解释的局限性在哪？还有什么没解答的？"
```
